# 웹 구조 디자인 스펙

> Phase 2 구현 전 확정된 웹 구조 설계 문서.

---

## 1. 페이지 구조 및 URL

| 페이지 | 파일 | URL |
|--------|------|-----|
| 인트로 | `index.html` | `/` |
| 공략 | `guide.html` | `/guide.html?class=전사&weapon=sword` |
| 에디터 | `editor.html` | `/editor.html` |
| 관리자 | `admin.html` | `/admin.html` |

- `index.html`에서 직업·무기 선택 완료 시 `guide.html?class=X&weapon=Y` 로 이동
- `guide.html` 진입 시 해당 (직업+무기)의 `isMain: true` 빌드를 자동 로드
- 빌드 전환 시 URL을 `?class=X&weapon=Y&build=sword_전사_1` 로 pushState 업데이트 (공유·북마크 가능)
- `isMain: true` 빌드가 없으면 빌드번호 0번(`_0`)을 기본 로드

## 2. 인트로 페이지 (`index.html`)

### 흐름
```
직업 선택 화면 → (선택) → 슬라이드 전환 → 무기 선택 화면 → (선택) → guide.html
```

### 직업 선택 화면
- 풀스크린 (모바일 viewport 전체)
- 5개 가로 카드 세로 나열, 스크롤 없음 (화면에 딱 맞춤)
- 카드 구성: 왼쪽 캐릭터 이미지 마스크 영역 + 중앙 직업명
- 상단: 사이트 타이틀 "MyheroGuide"

### 무기 선택 화면
- 풀스크린, 카드 스타일 동일
- 무기 수가 많으므로 세로 스크롤 가능
- 상단: `← 전사` (뒤로가기 버튼 + 현재 선택 직업명 표시)
- 카드 구성: 왼쪽 무기 이미지 마스크 + 중앙 무기명

### 전환 애니메이션
- 다음 단계 진입: 왼쪽으로 슬라이드
- 뒤로가기: 오른쪽으로 슬라이드

## 3. 공략 페이지 (`guide.html`)

### 상단 네비게이션 (고정, 2줄)
```
[검 · 전사]  [딜특화 ▾]    ← 1줄: 컨텍스트 바
[장비 ▾]                   ← 2줄: 탭 드롭다운
```
- **컨텍스트 바**: 무기종류·직업 레이블(고정 텍스트) + 빌드 전환 드롭다운
- **탭 드롭다운**: 13개 탭 직접 이동 (`<select>` 방식)

### 스와이프 제스처
| 방향 | 동작 |
|------|------|
| 좌우 스와이프 | 빌드 전환 (같은 무기+직업 내 빌드 순환) |
| 상하 스와이프 | 탭 전환 (13개 탭 순환) |

- 드롭다운과 스와이프 모두 동일한 상태 변경 (중복 제공)

### 콘텐츠 영역
- 각 탭의 내용이 화면 안에 모두 들어오도록 레이아웃 설계 (세로 스크롤 없음)
- 탭·빌드 전환 시 슬라이드 애니메이션

### 빌드 JSON 스키마 변경
기존 공통 메타 필드에 `isMain` 추가:
```json
{
  "buildId": "sword_전사_0",
  "buildName": "딜특화",
  "isMain": true,
  "status": "approved",
  ...
}
```

## 4. 미결 항목 (구현 시 결정)

- 직업·무기 이미지 실제 URL (운영자 제공 예정)
- 무기 종류 전체 목록 및 개수
- 빌드가 1개뿐인 경우 좌우 스와이프 비활성화 처리 방식
- 탭이 "준비 중"(은하장비)일 때 스와이프 건너뜀 여부
